## 1. SIMD 阶段的关注重点

在 SIMD 实验中，重点关注的是**对距离计算速度的优化**，即加快向量距离函数 \(\delta(x,y)\) 的计算。

ANN / ANNS 中常见的距离包括：

### 1.1 欧几里得距离

\[
\delta(x,y)=\sum_{i=1}^{d}(x_i-y_i)^2
\]

含义：逐维计算两个向量的差值平方，再求和。

### 1.2 内积距离

\[
\delta(x,y)=1-\sum_{i=1}^{d}x_i y_i
\]

含义：先计算两个向量的内积，再用 \(1-\text{inner product}\) 作为距离。

### 1.3 朴素 SIMD 优化

朴素的 SIMD 优化方式是直接对上述距离公式使用 SIMD 指令进行加速：

- 对欧几里得距离：向量化加载多个维度，批量计算 \((x_i-y_i)^2\)，最后归约求和。
- 对内积距离：向量化加载多个维度，批量计算 \(x_i y_i\)，最后归约求和。

也就是说，SIMD 阶段最基础的任务是把逐维 scalar 距离计算改成一次处理多个维度的向量化计算。

## 2. Vector Quantization，VQ

除朴素 SIMD 距离计算外，第 3.1 节还提出一种更好的距离计算优化方向：**Vector Quantization，VQ，向量量化**。

VQ 在本节中分为两类：

1. **Scalar Quantization，SQ**
2. **Product Quantization，PQ**

VQ 的核心思想是：不再总是直接使用原始 32-bit float 向量做完整距离计算，而是先把向量压缩或编码，再用更快的数据类型、查表或近似距离计算来加速查询。

---

## 3. SQ：Scalar Quantization

### 3.1 SQ 的动机

原始向量通常每一维都用一个 32-bit 浮点数存储。

如果使用 128-bit SIMD 寄存器加速计算：

- 对 float32：一次最多只能计算 4 个浮点数；
- 对 uint8：一次可以计算 16 个 8-bit 无符号整数。

因此，SQ 的思路是：

> 将向量的每一个维度从 32-bit float 量化为 8-bit unsigned integer，从而让 SIMD 寄存器一次处理更多维度。

### 3.2 SQ 带来的 SIMD 变化

| 数据表示 | 每维大小 | 128-bit SIMD 一次可处理数量 |
|---|---:|---:|
| float32 | 32 bit | 4 个 float |
| uint8 | 8 bit | 16 个 uint8 |

也就是说，在 128-bit SIMD 下，SQ 把单次 SIMD 可处理的元素数量从 4 个 float 提升到 16 个 uint8。

### 3.3 SQ 的要求与注意点

根据第 3.1.1 节，SQ 属于对距离计算进行优化的 VQ 方法之一。实验中如果采用 SQ，需要围绕以下点展开：

- 说明原始 float32 存储与 uint8 量化存储的区别；
- 说明 128-bit SIMD 下 float32 与 uint8 的并行宽度差异；
- 说明 SQ 通过量化降低存储和计算开销；
- 注意 SQ 会引入量化误差，因此不能只报告 latency，还应结合 recall 分析；
- 如果采用 coarse search + rerank，需要说明候选数量变化对 latency 和 recall 的影响。

## 4. PQ：Product Quantization

### 4.1 PQ 的基本流程

第 3.1.2 节以 \(N\) 条 128 维向量为例说明 PQ。

每条向量由 128 个浮点数组成：

\[
(x_0,x_1,\ldots,x_{126},x_{127})
\]

PQ 流程分为三步。

---

### 4.2 第一步：向量切分

首先对所有向量进行切分。

示例中把 128 维向量切成 4 段：

| 段编号 | 维度范围 |
|---|---|
| 第 1 段 | \((x_0,\ldots,x_{31})\) |
| 第 2 段 | \((x_{32},\ldots,x_{63})\) |
| 第 3 段 | \((x_{64},\ldots,x_{95})\) |
| 第 4 段 | \((x_{96},\ldots,x_{127})\) |

也就是说，每条 128 维向量被切成 4 个 32 维子向量。

---

### 4.3 第二步：对子空间分别聚类

把 4 个子段看作 4 个独立子空间，并分别对每个子空间中的子向量进行聚类。

对于 \(N\) 条向量：

- 第 1 个子空间聚类对象是 \(N\) 个 \((x_0,\ldots,x_{31})\)；
- 第 2 个子空间聚类对象是 \(N\) 个 \((x_{32},\ldots,x_{63})\)；
- 第 3 个子空间聚类对象是 \(N\) 个 \((x_{64},\ldots,x_{95})\)；
- 第 4 个子空间聚类对象是 \(N\) 个 \((x_{96},\ldots,x_{127})\)。

每一段聚类得到 256 个类中心。

因此示例中总共保存：

\[
256 \times 4 = 1024
\]

个类中心，并按顺序编号。

---

### 4.4 第三步：原始向量量化编码

对原始向量进行量化编码：

- 每个子段用距离最近的聚类中心编号表示；
- 原来的 32 个 float 子向量被替换为一个整数 ID；
- 4 个子段分别得到 4 个整数 ID。

示例说明：

如果第一条向量的第一段 \((x_0,\ldots,x_{31})\) 距离第一段的第 124 个类中心最近，则第一段 32 个浮点数被替换为整数 `124`。第二、三、四段同理。

最终，一条由 128 维 float 组成的向量被压缩成 4 个整数。

## 5. PQ 查询阶段的两种距离计算策略

当一个查询向量到达时，需要计算查询向量与 PQ 压缩后的 Base 向量之间的距离。第 3.1.2 节列出两种策略：

1. **SDC：Symmetric Distance Computation，对称距离计算**
2. **ADC：Asymmetric Distance Computation，非对称距离计算**

---

## 6. SDC：对称距离计算

### 6.1 SDC 的查询处理方式

SDC 会对查询向量执行与 Base 向量相同的量化编码。

示例流程：

1. 将查询向量切分为 4 个子向量；
2. 分别计算每个查询子向量与对应子空间中 256 个类中心的距离；
3. 每个子向量用最近类中心的 ID 编码；
4. 例如查询向量可能被编码为：

```text
(25, 82, 10, 254)
```

### 6.2 SDC 的距离计算方式

查询向量与 Base 向量之间的距离被转化为查表累加：

- 查阅预先计算好的 \(256 \times 256\) 中心间距离表；
- 对 4 个子空间分别查表；
- 对 \(N\) 个 Base 向量执行 \(4 \times N\) 次查表累加。

### 6.3 SDC 的特点

SDC 的特点是：

- 查询向量也被量化；
- 查询和 Base 都在量化编码空间中比较；
- 需要维护中心间距离表；
- 距离计算可转化为查表累加；
- 在批量查询场景下，SDC 具有流水线并行潜力。

---

## 7. ADC：非对称距离计算

### 7.1 ADC 的查询处理方式

ADC 不对查询向量进行编码。

查询到达时，直接计算查询的原始子向量与各子空间类中心之间的精确距离。

示例中，查询向量被切成 4 个子向量，因此生成一张局部查找表：

\[
4 \times 256
\]

也就是每个子空间对应 256 个类中心距离。

### 7.2 ADC 的距离计算方式

查询距离通过 Base 编码在局部 LUT 中查表获得：

- 查询向量保持原始 float 子向量；
- Base 向量使用 PQ 编码；
- 每个 Base 向量通过其 code 在 \(4 \times 256\) 的 LUT 中查表；
- 对 \(N\) 个 Base 向量执行 \(4 \times N\) 次查表累加。

### 7.3 ADC 的优势

第 3.1.2 节明确指出，相比 SDC，ADC 具有显著优势：

- ADC 仅引入 Base 端的量化误差；
- 召回率更高；
- 无需维护庞大的中心间距离表；
- SDC 与 ADC 在实时查询，也就是单条触发时，总计算量相当；
- ADC 往往能获得更好的性能权衡。

---

## 8. SDC 的流水线并行探索

第 3.1.2 节在“更多探索”中指出：在处理批量查询场景下，SDC 方案具有流水线优化潜力。

原因：

- SDC 涉及查询向量的量化编码；
- 查询编码阶段和搜索查表阶段可以重叠。

可以尝试的生产者-消费者模型：

1. 使用独立线程对下一批次查询向量进行异步编码；
2. 搜索线程并行执行当前批次的查表累加；
3. 甚至可以尝试分配独立线程进行结果重排。

这种流水线设计的目标是：

> 掩盖查询编码阶段的延迟。

该方向被明确列为后续实验中值得尝试的进阶方向。

## 9. 3.1.3 更多探索

第 3.1.3 节说明，这部分技术难度较大，学有余力的同学可以自主探索。

### 9.1 FastScan

资料指出，单纯使用前述算法仍然不能满足如今大规模检索的性能需求。

目前 PQ 通常会与一种叫做 **FastScan** 的技巧结合使用。

FastScan 首次在论文：

```text
Cache locality is not enough: high-performance nearest neighbor search with product quantization fast scan
```

中提出。

要求或建议：

- 可以参考该论文；
- 可以参考 FastScan 实现细节；
- 可以将 FastScan 作为更高难度的自主探索方向。

### 9.2 更多 VQ 算法

资料还建议探索更多 VQ 算法，包括：

1. **Optimized Product Quantization，OPQ**
2. **RaBitQ: Quantizing High-Dimensional Vectors with a Theoretical Error Bound for Approximate Nearest Neighbor Search**

这些方向属于自主探索，不是 3.1 节基础说明中的朴素 SIMD 主线，但它们确实出现在第 3.1.3 节中，因此本文件保留。
